# План запросов (PROMPT_PLAN.md) — NUTRI TRADE LTD

Этот документ содержит стратегию итеративной разработки проекта лендинга для B2B-поставщика Nutri Trade Ltd.

## Этап 1: Структура и правовой фундамент
- **Цель:** Создать семантическую структуру сайта и обязательные юридические страницы.
- **Запросы:**
    1. Спроектировать структуру `index.html` (Hero, О компании, Продукты, Процесс, Преимущества, Контакты).
    2. Создать черновики `privacy.html` и `cookies.html` с учетом требований GDPR для Болгарии.
    3. Разработать базовую CSS-сетку (Flexbox/Grid), соответствующую дизайн-системе (цвета: крем, шоколад, оранжевый).

## Этап 2: Контент и визуальное оформление
- **Цель:** Наполнение секций контентом из клиентского ТЗ и базовая стилизация.
- **Запросы:**
    1. Интеграция текста из `01-landing-client-content.pdf` в соответствующие блоки `index.html`.
    2. Оформление карточек продуктов и элементов процесса работы (timeline).
    3. Применение глобальных стилей: шрифты, отступы, адаптивные брейкпоинты для мобильных устройств.

## Этап 3: Интерактив и форма сбора лидов
- **Цель:** Добавление функциональности, соответствующей критериям приемки.
- **Запросы:**
    1. Разработка валидации формы (проверка email, обязательность чекбокса GDPR).
    2. Реализация состояний формы (success/error).
    3. Добавление скриптов для меню-бургера и cookie-баннера (Intersection Observer для анимаций).

## Этап 4: Финализация и документация
- **Цель:** Проверка соответствия проекта всем требованиям, создание документации.
- **Запросы:**
    1. Проверка семантики HTML и доступности (Accessibility).
    2. Генерация проектных файлов: `TASK.md`, `README.md`, `AI_USAGE.md`.
    3. Настройка конфигурации `.cursor/rules/project.mdc` для фиксации ограничений проекта (цвета, запрет фреймворков).
